一，软件介绍
JCrage是crage的Java GUI，可以利用内部的自动侦测模式配合plugin.xml来实现一定程度的参数自动填写。
使用前请确保已安装Java Runtime Environment(1.6版本)，下载见http://java.sun.com/javase/downloads/index.jsp。



二，操作步骤(经典)
1，选择"输入"标签页
2，选择"源封包"或"源目录"。
3，若"选择CUI"没有被自动激活(没有打勾)，说明不需要指定插件，请到7；否则说明需要指定插件或者索引或者特殊参数，请到4。
4，选择CUI插件。
5，若"必要索引"没有被自动激活(没有打勾)，说明不需要指定索引参数，请到6；若被自动激活且被自动填充，说明需要自动侦测模式已发现索引参数，请到6；否则说明自动侦测模式被没有发现索引参数，请手动选择索引参数。
6，若"特殊参数"没有被自动激活(没有打勾)，说明不需要指定特殊参数，请到7；若被自动激活且被自动填充，说明需要自动侦测模式已发现特殊参数中的Game参数，请到7；否则说明自动侦测模式已发现特殊参数中的其它参数，请手动选择正确的参数。
7，点击"列队"。
8，选择"队列"标签页。
9，点击"开始"。



三，界面介绍(详细)
(1)，输入标签页
	基本选项框
		源封包：手动选择封包路径，与源目录是二选一的必选项，对应crage参数是-p。
		源目录：手动选择目录路径，与源封包是二选一的必选项，对应crage参数是-d；最好为封包所在当前路径，否则自动侦测模式会失效。
		必要索引：手动或自动选择索引路径，为可选项，对应crage参数是-l；若已选择CUI并激活自动侦测模式，将会自动显示可能的索引路径(可能错误)。
	高级选项框
		输出目录：手动选择输出路径，为可选项，对应crage参数是-o。
		选择CUI：手动选择CUI，为可选项，对应crage参数是-u；若激活自动侦测模式，将会列出所有符合封包后缀名的CUI。
		阅读文档：若已选择CUI，将会打开对应存在的文档。
		特殊参数：手动或自动输入特殊参数，为可选项，对应crage参数是-O；若已选择CUI并激活自动侦测模式，将会自动列出所有可能的特殊参数。
	杂项选项框
		忽略错误：为可选项，对应crage参数是-F，实际为不可选项，程序强制使用此参数。
		提取原始资源：为可选项，对应crage参数是-r。请不要选中只解包不提取，否则会生效。
		只解包不提取：为可选项，对应crage参数是-n。请不要选中提取原始资源/+指定资源提取，否则会生效。
		查看详情：为可选项，对应crage参数是-v。
		检测CUI版本：为可选项，对应crage参数是-I。请先选择CUI，否则会失效，同时其他选项全部失效。
		指定资源提取：为可选项，对应crage参数是-f。请不要选中只解包不提取，否则会生效。
	其它
		刷新：刷新当前输入标签页。
		列队：列队生成的命令到队列标签页。

(2)，队列标签页
	右键菜单
		开始：执行队列表格中的状态为等待的选中命令。
		清除：清除队列表格中的选中命令。
		上移：上移队列表格中的选中命令。
		下移：下移队列表格中的选中命令。
		状态：改变队列表格中的选中命令的状态。若选择等待，则执行；若选择延迟，则不执行。
	其它
		开始：执行队列表格中的所有状态为等待的命令。
		保存：保存队列表格中的所有命令到批处理文件(crage.bat)。
		清空：清空队列表格中的所有命令。
		日志模式：若选择独立文件，则一个任务记录到日志目录(logs)下的一个日志文件(JobFile_xxx.log)；若选择合并文件，则所有任务记录到一个日志文件(job.log)。

(3)，日志标签页
	右键菜单
		复制：复制选中的树项内容到剪贴板。
		保存：复制选中的树项内容到文本文件(InfoFile_xxx.txt)。

(4)，进度监视窗口
	终止：当点击终止时，会暂停当前命令；当确认终止时，会终止当前命令。



四，程序清单
JCrage.jar		必备，Java GUI
crage.exe		必备，专属JCrage的crage.exe升级版
utility.dll		必备，专属JCrage的utility.dll升级版
plugin.xml		必备，参数自动化配置文件
JCrage_readme.txt	必备，说明文件
error.log		可选，错误日志
job.log			可选，任务日志
